Go Read/Write-only channels

  • golang
  • cuncurrency
  • channels

If we declare a chan in function argument or return types like <-chan, it will be receive-only, meaning that its users cannot write something to that channel This for example, leads to an error: func main() { ch := random() ch <- 55 fmt.Println(<-ch) } func random() <-chan int { ch := make(chan int, 1) ch <- 24 return ch } invalid operation: cannot send to receive-only channel ch (variable of type <-chan int) The same goes to function arguments.
